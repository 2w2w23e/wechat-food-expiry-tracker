# Decisions - 2026-06

## 2026-06-15: Adopt layered RepoMind OS migration

Status: accepted.

Decision maker: user.

Context:

The repository already had an older AI collaboration system under `docs/`, including role prompts, Codex workflow rules, document ownership rules, and tool or Skill policies.

RepoMind OS was introduced afterward under `.ai-governance/`.

Decision:

Use layered migration.

Consequences:

- `.ai-governance/` becomes the highest governance layer for the repository.
- Existing `docs/*` AI collaboration documents remain as historical governance documents and import evidence.
- Old role prompts are not automatically converted into RepoMind OS role files.
- A full role library should not be created immediately.
- Verified project state should first be written into `.ai-governance/PROJECT_STATE.md`.
- Current bootstrap status should be written into `.ai-governance/handoff/CURRENT.md`.
- Old foundation documents should be synchronized only after user-approved scope is clear.

Not decided yet:

- Whether old roles should be migrated into `.ai-governance/roles/`.
- Whether V0 should be marked complete or code-complete candidate pending manual verification.
- Whether the next product direction should be V0 acceptance/release polish or V1 planning.

Stop boundaries:

- Do not write implementation code as part of this decision.
- Do not ask Codex to modify source files during bootstrap.
- Do not start V1 implementation from this decision alone.
- Do not introduce cloud development, barcode, OCR, AI, reminders, new dependencies, or new Skills without separate user approval.
