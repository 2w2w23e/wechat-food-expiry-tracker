
# 食期管家 / WeChat Food Expiry Tracker

食期管家是一个微信小程序项目，用于记录和管理食品的生产日期、保质期、最终可食用日期、保存方式和备注，帮助用户按到期时间管理食品，减少遗忘、浪费和误食风险。

## 项目目标

本项目计划实现：

- 手动新增、编辑、删除食品信息
- 支持输入“生产日期 + 保质期”自动计算最终可食用日期
- 支持用户直接输入最终可食用日期
- 记录保存方式，例如常温、冷藏、冷冻、避光、干燥、开封后冷藏等
- 按最终可食用日期自动排序
- 显示已过期、今日到期、即将到期、正常等状态
- 支持到期提醒
- 后续支持拍照识别食品包装信息，并在用户确认后保存

## MVP 优先级

第一阶段优先完成：

1. 项目初始化
2. 食品数据模型
3. 手动录入页面
4. 日期计算工具函数
5. 食品列表页面
6. 到期状态显示
7. 基础排序
8. 本地数据或云数据库保存

第二阶段再考虑：

1. 云数据库用户隔离
2. 云函数
3. 到期提醒
4. 拍照识别入口
5. OCR 原文展示
6. 识别结果确认页

## 技术方向

默认技术方向：

- 微信小程序原生开发
- WXML
- WXSS
- JavaScript 或 TypeScript
- 微信云开发 / CloudBase
- 云数据库
- 云函数

## 重要原则

- 最终可食用日期 `expiryDate` 是排序和提醒的核心字段。
- OCR 和 AI 识别结果不能直接入库，必须经过用户确认。
- API Key、OCR 密钥、OpenAI 密钥等敏感信息不能写入小程序前端代码。
- 涉及 OCR、AI、订阅消息、云数据库权限的能力，应放在云函数或服务端处理。
