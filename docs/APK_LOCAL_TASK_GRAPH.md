# APK 本地版任务图与调度计划

日期：2026-07-05

## 1. 产品边界

当前主线只做 Android 本地 APK。

当前已纳入本地 APK 的能力包括商品码识别、商品信息查询、本地提醒、系统通知和已用完归档；本任务图不把这些能力误写成“当前不做”。

不进入以下方向：

- 应用商店发布
- 云数据库
- 账号体系
- 多设备同步
- OCR / AI 自动入库
- 医疗、营养、绝对食品安全判断

核心约束：

- `expiryDate` 仍是排序、状态和提醒的标准字段。
- APK 版本升级不能导致本地食品数据丢失。
- 条码、OCR、AI 或外部识别结果必须经过用户确认后才能保存。
- 新字段必须向后兼容旧 JSON。

## 2. 同类产品参考结论

参考 NoWaste、Food Expiration Dates、KitchenPal、Pantry Check、BEEP、Fango 等同类项目后，后续功能重点不应先做“大而全”，而应优先降低日常维护成本：

- 录入更快：条码历史、复制上次、快速消耗。
- 查看更清楚：冰箱/冷冻/常温/柜子位置、临期时间线、今日待处理。
- 提醒更准确：全局提醒设置、单品提醒覆盖、开封后保质期。
- 数据更可靠：schema 版本、迁移、备份、导出导入、损坏恢复。
- 测试更稳定：纯 Java 逻辑测试、构建验证、升级数据回归。

## 3. 调度算法

任务图使用 DAG。有向边 `A -> B` 表示 B 必须等待 A 完成。

调度采用：

1. Kahn 拓扑排序维护入度。
2. 所有入度为 0 且未完成的节点进入任务池。
3. 对任务池按关键路径长度排序，优先调度“下游最长”的节点。
4. 同关键路径长度时，优先短任务和低冲突任务。
5. 并行上限默认 3 个子智能体。
6. 每个子智能体必须拥有互不重叠的写入范围。
7. `VERIFY-ALL` 是显式测试门禁节点，所有实现任务完成后才允许执行。

关键路径长度定义：

```text
critical_path(node) = estimate(node) + max(critical_path(successor))
```

当前第一轮无入度任务池：

```text
PLAN-001
TEST-001
DATA-001
DOC-001
```

## 6. 数据不丢失策略

版本升级必须遵守：

1. 读取旧 JSON 时永远先保留原始字符串。
2. 迁移失败时不能覆盖原存储。
3. 保存新结构前写入最近备份。
4. 新字段必须有默认值。
5. 导入必须先预览和确认。
6. `isFinished`、`finishedAt`、未来的 `location`、`openedDate` 等字段都必须向后兼容。

## 7. 第一轮并行分配

第一轮使用 3 个并行槽：

- 主脑本地：`PLAN-001`
- 子智能体 A：`TEST-001`
- 子智能体 B：`DATA-001`
- 子智能体 C：`DOC-001`

完成任一节点后，按 Kahn 算法释放后继节点并重新计算任务池。

## 9. 视觉 QA 门禁

用户明确要求 APK UI 不能只靠代码推断验收。所有涉及界面、安装、扫码、权限、弹窗、列表布局、筛选入口、提醒设置和数据导入导出的任务，都必须在真实 Android 设备或可启动模拟器上完成截图或录屏验证。

当前执行环境状态：

- 已创建并启动 `Medium_Phone` AVD，`adb devices` 可识别 `emulator-5554`。
- 2026-07-05 已完成首页、食品卡片、新增表单、日期选择、详情、更多操作、提醒设置和 Gradle 覆盖安装首页 smoke 截图。
- 视觉 QA 仍为 `PARTIAL`：摄像头扫码、图库识别、手动条码输入、商品查询确认保存和编辑表单截图仍需真机或稳定模拟器复测。
- 因此 `VERIFY-ALL` 仍不能标记为全量 PASS；涉及 UI 的后续任务必须继续补截图或录屏。

视觉验证解除阻塞后，最少要覆盖：

- 首页食品卡片：新增位置、快速消耗、补货、编辑、已用完、删除按钮布局。
- 新增 / 编辑食品表单：位置、开封日期、开封后保质期、提醒设置相关字段。
- 食品详情：开封信息、更多操作、复制食品、剩余归 0、补货弹窗。
- 筛选入口：状态、分类、位置组合筛选。
- 提醒卡片：全局提醒开关、提前天数、今日提醒时段。
- 条码入口：相机扫码、图库识别、手动输入、查询结果确认后保存。

## 10. 同类项目参考来源

本轮任务计划参考了这些公开项目或应用页面，取其通用信息架构，不复制其实现：

- Grocy：https://grocy.info/ ，自托管家庭库存管理，强调条码录入、库存、到期提醒和移动端配套。
- NoWaste：https://play.google.com/store/apps/details?id=com.khcreations.nowaste ，冰箱 / 冷冻 / 储藏列表、条码添加、按过期日期排序、分类和位置筛选。
- Pantry Check：https://pantrycheck.com/ ，按单品跟踪到期日、到期提醒、条码、位置和商品照片。
- KitchenPal：https://play.google.com/store/apps/details?id=fr.icuisto.icuisto ，位置分区、数量、到期日、提醒、条码和导出能力；其中云同步、家庭共享、营养 / 饮食建议不纳入当前本地 APK。
