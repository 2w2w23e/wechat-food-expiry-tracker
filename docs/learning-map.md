这个文件用于记录开发“食期管家”过程中需要学习的知识。目标不是只完成项目，而是在开发过程中逐步理解微信小程序开发、GitHub 协作、Codex 使用和云开发。

## 1. 微信小程序基础

需要理解：

- 小程序项目结构
- `app.json` 的作用
- 页面目录结构
- WXML 是什么
- WXSS 是什么
- JavaScript / TypeScript 在小程序中做什么
- 页面生命周期
- 数据绑定
- 事件绑定
- 组件拆分

学习目标：

- 能看懂一个页面由哪些文件组成
- 能理解页面数据如何显示到界面上
- 能理解用户点击、输入如何触发逻辑

## 2. 食期管家的核心业务知识

需要理解：

- 什么是生产日期
- 什么是保质期
- 什么是最终可食用日期
- 为什么最终可食用日期是排序和提醒核心字段
- 为什么支持手动输入最终可食用日期
- 为什么 OCR 结果必须由用户确认

学习目标：

- 能解释 `expiryDate` 的作用
- 能判断日期计算是否正确
- 能识别 OCR 自动入库的风险

## 3. 日期计算

需要理解：

- 天、月、年的保质期计算差异
- 闰年
- 月底日期
- 无效日期
- 用户手动覆盖最终日期
- 开封后保质期与原始保质期的关系

学习目标：

- 能写出日期计算的测试用例
- 能看懂日期计算工具函数
- 能判断哪些情况需要用户手动确认

## 4. GitHub 协作

需要理解：

- repository 是什么
- commit 是什么
- branch 是什么
- pull request 是什么
- README.md 的作用
- AGENTS.md 的作用
- docs 目录的作用

学习目标：

- 能把本地修改提交到 GitHub
- 能看懂一次 PR 改了什么
- 能通过文档让 Codex 更稳定地工作

## 5. Codex 使用

需要理解：

- Codex 适合做什么
- 为什么任务要小
- 为什么要写清楚涉及文件
- 为什么要写验收标准
- 为什么要写禁止修改范围
- 如何检查 Codex 的改动

学习目标：

- 能把一个需求拆成 Codex 任务
- 能审查 Codex 改动
- 能避免 Codex 一次性大改项目

## 6. 微信云开发

需要理解：

- 云数据库是什么
- 云函数是什么
- 用户 openid 是什么
- 数据权限是什么
- 为什么密钥不能放前端
- 为什么 OCR / AI 识别要放云函数或服务端

学习目标：

- 能理解食品数据如何保存
- 能理解用户数据如何隔离
- 能知道哪些逻辑不能放在小程序前端

## 7. OCR 和 AI 识别

需要理解：

- OCR 是什么
- OCR 原文是什么
- 信息抽取是什么
- 置信度是什么
- 为什么识别结果可能出错
- 为什么必须有确认页

学习目标：

- 能设计 OCR 结果确认流程
- 能判断哪些字段可以自动填充
- 能判断哪些字段必须提示用户检查
