# 关键帧 OCR 统一架构与公开样本验证报告

## 2. 统一架构

```text
相机帧 / 视频帧 / 图库图片
  -> 图库 EXIF 方向校正 + 帧质量测量
  -> 原图、日期区域与 OpenCV 低对比增强变体
  -> ML Kit 中文 / 拉丁文字识别 + ML Kit 条码识别
  -> PP-OCRv6 对候选日期行和日期带复识
  -> 带帧号、区域框、模型来源和质量分的字段证据
  -> 日期门禁 + 商品名冲突回退
  -> 关键帧选择 + 跨帧候选稳定
  -> 用户查看关键帧和推荐候选
  -> 用户采用结果后填入可编辑新增表单
  -> 用户主动保存
```

### 2.1 模型与规则分工

| 组件 | 当前职责 | 明确不负责 |
| --- | --- | --- |
| ML Kit Text Recognition v2 Chinese / Latin | 通用包装文字、中文标签、拉丁字母、数字及文字框 | 不单独决定最终字段 |
| ML Kit Barcode Scanning | 本地条码识别，为商品查询和商品名优先级提供证据 | 不自动保存商品信息 |
| OpenCV | 保留原图并生成 CLAHE、背景差分、二值化等低对比变体 | 不把增强伪影当成唯一证据 |
| `PP-OCRv6_rec_small` | 对已经检测或裁剪的日期文字行、日期带进行补充复识 | 不做整图文字检测、方向分类或商品名决策 |
| 日期解析与证据策略 | 标签关联、日期归一化、生产/有效期分配、未来生产日期拦截 | 不从缺少年份或非法数字中强行造日期 |
| 关键帧与稳定器 | 合并帧质量、字段区域、语义、模型来源和跨帧重复证据 | 不把启发式分数称为校准概率或用户确认 |

### 2.2 关键帧和可见证据

`RecognitionFrameSelector` 最多保留 3 个关键帧。评分同时考虑清晰度、曝光平衡、反光比例、文字覆盖、OCR 置信和视频时间位置；近重复帧只保留更强者，最终组合还优先覆盖商品名、生产日期、有效期和条码等不同字段。

`RecognitionEvidence` 为商品名、生产日期、有效期和条码保存归一化区域框、来源引擎、模型置信和融合置信。识别页显示关键帧缩略图和字段框，用户可以看见推荐值来自哪一帧、哪一块区域。页面中的“高/中/低”是规则融合等级，不是统计学概率；单帧证据会被限制上限，跨帧重复和来源互补才提高稳定度。

### 2.3 确认后填表，不自动保存

`DateOcrScanActivity` 的“采用所选结果”只通过 Activity result 返回商品名、条码和日期草稿；`MainActivity` 收到草稿后调用新增表单，并提示用户检查。识别流程本身不调用食品存储写入，只有用户在可编辑表单中主动保存后才进入食品列表。`expiryDate` 仍是保存后排序和提醒的规范字段。

## 3. 日期与商品名门禁

### 3.1 EXIF 与多格式日期

- 图库文件和内容 URI 在 OCR 前读取 EXIF 方向，处理正常、水平/垂直翻转、90/180/270 度旋转及转置方向，避免“像素已旋转但 OCR 仍按原方向读取”。
- 日期支持 `YYYY-MM-DD`、`YYYY/MM/DD`、`YYYY.MM.DD`、中文年月日、`DD/MM/YYYY` 等四位年后置格式、两位年格式、6/8 位紧凑数字、英文月份，以及带明确有效期语境的年月或月/年。
- 月级有效期只生成按当月末日归一化的可编辑候选，并必须提示“包装标到月，按月末”。
- 只有月日、没有年份的文本不补猜年份。510-Date 的 `10.01` 和 ExpDate `test_00616` 的 `04.10` 均按安全拒绝处理。

### 3.2 未来生产日期门禁

日期可以保留在原始 OCR 文本中供用户查看，但晚于设备当天的值不会被预选为生产日期。无标签的单个未来日期只能作为较弱的有效期候选；明确写成未来生产日期的值也会在字段证据策略中被移除，由此计算出的有效期候选同步失效。本地测试覆盖“未来生产日期不预选”和“不得由未来生产日期计算有效期”两条门禁。

### 3.3 商品名冲突回退

商品名优先级是：稳定条码商品库名称、明确商品名标签、可信包装文字。无可靠品牌共识时，不拼接或猜测冲突品牌，而是退回画面中确实可见的食品品类短语，例如“饮用纯净水”“饮用净水”“天然水”“纯牛奶”。不同水类品类不会因文字相似而错误合并。

这类回退只说明“画面上可见的食品类型足以作为可编辑候选”，不说明品牌或完整商品名已经识别成功。

## 4. 公开样本结果

### 4.1 510-Date

固定代表样本为 `0066`、`0140`、`0217`、`0393`、`0394`、`0407`、`0463`、`0477`、`0482`、`0508`，覆盖多种分隔符、年月顺序、低对比和无年份文本。

- 终态：9 个完整日期准确。
- `0508` 只有 `10.01`，没有年份，系统不填入日期，属于安全拒绝。
- 主要本地证据：`.local/public-validation/avd-date510-results-v2.json`、`.local/public-validation/date0140-final3.png` 和 `.local/public-validation/510-Date/visual-selection.json`。

`date0140-final3.png` 是晚于批量 JSON 的针对性重跑证据，显示 `2021-09-24`。因此不能只读取较早的批量中间结果来代表终态。

### 4.2 ExpDate

固定代表样本为 `test_00024`、`test_00074`、`test_00188`、`test_00205`、`test_00249`、`test_00330`、`test_00541`、`test_00562`、`test_00573`、`test_00616`。

- 终态：9 个完整日期准确，1 个无年份样本安全拒绝。
- `test_00562`：识别并归一化为 `2024-01-12`。
- `test_00249`：根据可见 `Best Before` / 有效期语义，只保留有效期 `2021-04-03`，不再同时填成生产日期。
- `test_00616`：只有 `04.10`，没有年份，不填入日期。
- 其余 7 个代表样本的完整日期与冻结真值一致。
- 主要本地证据：`.local/public-validation/ExpDate/extracted/Products-Real/evaluation/qa-selection-24.json`、`.local/public-validation/avd-expdate-results-v2.json`、`.local/public-validation/exp562-final.xml`、`.local/public-validation/exp249-fixed.xml` 及同目录终态截图。

ExpDate 来源页未给出明确数据许可，本地结果只用于研究和失败诊断，不提交原图，也不作为可公开复用的正式金标指标。

### 4.3 Open Food Facts 中文图片

Open Food Facts 的商品名是社区元数据，未必与某一张图片严格对应；本轮只核对“是否选择画面可见且不冒进的食品品类”，不计算准确率。

| 示例 | 当前终态 | 证据 |
| --- | --- | --- |
| 怡宝图片 | 冲突品牌不强猜，退回 `饮用纯净水` | `.local/public-validation/final-yibao.xml` |

这些结果证明了 4 个策略示例的终态行为，不证明 Open Food Facts 中国商品全量、所有品牌或所有拍摄条件均可准确识别。

## 5. APK 模型边界

当前构建产物 `apk/app/build/outputs/apk/debug/app-debug.apk` 为 `212,555,545` 字节。源码资源和 APK ZIP 条目核对结果如下：

- APK 包含 `assets/PP-OCRv6_rec_small.onnx`，大小 `21,234,383` 字节。
- 本机研究目录另有 `.local/models/ppocrv5-server/PP-OCRv5_server_rec_infer.onnx`，大小 `84,505,505` 字节，以及本地字典文件。
- `.local/` 被 Git 忽略；当前 APK ZIP 中没有 `PP-OCRv5_server_rec_infer.onnx` 或对应 server 字典。
- APK 仍会包含 ML Kit bundled 模型和 ONNX Runtime/OpenCV 原生库；“不包含 server 模型”只针对本地下载的 PP-OCRv5 server 研究模型，不表示 APK 没有任何 OCR 模型。

## 6. 真实限制

- 没有执行 510-Date 或 ExpDate 全量评测，也没有建立随机抽样、置信区间、召回率或误报率。
- 代表样本经过固定选择，用于覆盖格式和失败类型，不是统计学随机样本。
- Open Food Facts 只做 4 个弱标注中文图片的定性策略检查。
- 本轮没有真实 Android 手机相机验收，不声称自动对焦、曝光、反光、运动模糊、连续扫描或低端设备性能已通过。
- PP-OCRv6 是日期行复识补充，不是完整 PaddleOCR 检测流水线；文字框遗漏时仍可能没有可复识的日期行。
- EXIF 校正解决静态图片方向问题，不能修复失焦、遮挡、严重透视或已经丢失的像素。
- 所有结果仍需用户核对。识别候选和日期推断不构成食品安全、医疗或营养结论。

因此，当前可以诚实表述为：本地规则测试通过，两个固定公开日期子集各得到“9 个完整日期准确 + 1 个无年份安全拒绝”，4 个中文包装策略示例符合保守回退目标，当前 APK 未打包本地 server 研究模型；但 9 个月保质期视频仍失败，其他旧视频未在最终代码上全部重跑，Excel 覆盖/回滚与真实相机验收也未完成。当前状态为 `PARTIAL`，不可作为“全部样本通过”的验收发布版。
