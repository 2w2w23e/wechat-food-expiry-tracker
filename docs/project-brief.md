# 食期管家项目简报

## 1. 项目名称

食期管家

## 2. 项目类型

Android APK 本地版优先；微信小程序端封存在 `miniprogram/`

## 3. 项目一句话说明

食期管家是一个用于记录食品生产日期、保质期、最终可食用日期、保存方式和备注的食品库存管理应用。当前暂时转为 Android APK 本地版开发，已验收的小程序端源码封存在 `miniprogram/`。

## 4. 核心问题

用户在家庭、宿舍或个人生活中，经常会忘记食品什么时候买的、什么时候生产的、什么时候过期，以及应该如何保存。本项目希望用轻量方式帮助用户记录和提醒。

## 5. 核心用户

- 家庭用户
- 独居用户
- 宿舍用户
- 有囤货习惯的用户
- 希望减少食品浪费的人

## 6. MVP 功能范围

当前 APK 本地版的核心功能范围：

1. 手动新增食品
2. 编辑食品
3. 删除食品
4. 输入生产日期和保质期后自动计算最终可食用日期
5. 支持直接输入最终可食用日期
6. 记录保存方式和备注
7. 食品列表按最终可食用日期排序
8. 显示到期状态
9. 商品码扫描、图库识别和手动输入查询，保存前需要用户确认
10. 本地提醒计划、今日简报、今日到期提示和 Android 本地系统通知
11. 已用完归档和恢复

## 7. 后续功能范围

近期优先：

1. 本地数据 schema 版本和迁移
2. 旧 JSON 兼容读取和升级回归
3. 保存前备份、损坏恢复、导出导入
4. 纯 Java 逻辑测试和 APK 构建环境验证
5. 扫码、提醒和已用完归档的真机回归

暂缓评估：

1. 云数据库保存
2. 用户数据隔离
3. 多设备同步
4. 应用商店发布
5. OCR / AI 信息抽取

## 8. 非目标

当前阶段不做：

1. 不判断食品是否绝对安全可食用
2. 不做医疗或营养建议
3. 不做复杂社区功能
4. 不做电商比价
5. 不允许 OCR 结果跳过确认直接保存
6. 当前不做云存储、账号体系、多设备同步或应用商店发布计划

## 9. 核心字段

食品信息至少包括：

- 食品名称
- 生产日期
- 保质期
- 最终可食用日期
- 保存方式
- 数量
- 分类
- 备注
- 创建时间
- 更新时间

其中，最终可食用日期 `expiryDate` 是排序和提醒的核心字段。

## 10. 技术方向

当前优先采用：

- Android 原生本地版
- Java
- Android SDK 命令行构建
- debug APK

封存的小程序端保留：

- 微信小程序原生开发
- WXML
- WXSS
- JavaScript 或 TypeScript
- 微信云开发 / CloudBase
- 云数据库
- 云函数

这些小程序和云开发技术仅为归档能力说明，不代表当前 APK 主线启用云端开发。

## 11. 安全原则

- API Key、OCR 密钥、OpenAI 密钥不能放在小程序或 Android 客户端代码中。
- 需要密钥、账号或付费接口的 OCR、AI 和外部查询逻辑应放在服务端；当前 APK 本地版不新增云端能力。
- OCR 识别结果必须进入用户确认页面。
- 用户确认后才能保存识别结果。
