## 体验中发现并修复

- 大字体下识别页底部“填入表单”被固定高度截断：改为按字体比例扩展面板，同时限制最大高度保留预览。
- “已用完”筛选在横向状态列表后部不易找到：调整为首个状态入口。
- 恢复已用完食品后仍显示剩余 0：恢复时至少补回 1 个剩余数量。
- 编辑自动计算日期食品会静默改为手动来源，且日期输入变化时旧预览不更新：增加自动/手动状态，实时重算，并让日期选择器从当前自动日期开始。
- 下一次提醒显示未来提醒时间，却附带当前剩余天数：改为显示未来节点是“到期前几天”。
- Excel 只有警告时仍写“错误行详情”：统一为“问题行详情 / Excel 导入详情”。
- `SharedPreferences.commit()` 返回失败时新值仍可能进入进程缓存：`FoodStore` 在写入前保存主数据、迁移备份和三代滚动备份的完整快照，失败时恢复全部键；Activity 同时恢复最近一次成功食品快照。
- 用户在 Excel 中填写的日期通常是数字日期单元格：增加 `styles.xml` 日期格式识别与 1900/1904 日期系统转换。
- 同一批 Excel 行可能在同一毫秒获得重复 ID：改为 UUID，并在批次内保留已分配 ID 集合；`calculated` 日期来源与生产日期、保质期不一致时降级为可安全编辑的手动日期并提示警告。
- 旧版本本地 JSON 也可能含有不完整的 `calculated` 来源：加载迁移时保留原有效到期日、改为手动来源并写回，避免升级后只编辑名称就清空或改动日期。
- 固定模式未配置到期当天节点时仍承诺后续提醒：没有未来节点时明确显示“暂无后续提醒”。
