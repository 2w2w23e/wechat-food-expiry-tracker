## 当前版本

版本状态：进行中

最后更新角色：main-brain

## 当前 V0 任务边界

V0 的目标是完成一个可实际试用的手动版“食期管家”。用户可以手动录入食品，查看库存，看到临期 / 过期状态，按最终可食用日期排序，并完成基础筛选、统计和保存。

V0 当前仍不做：条形码扫描、商品信息查询、文字识别、智能识别、菜谱规划、复杂提醒、复杂看板、家庭权限、识别结果自动保存、前端密钥存储、新依赖或云开发。

## V0 内部开发顺序

1. 核心数据与日期计算基础。
2. 食品基础数据结构与本地模拟数据。
3. 食品列表展示、`expiryDate`（最终可食用日期字段）排序和到期状态显示。
4. 手动新增食品。
5. 食品卡片详情查看。
6. 编辑和删除食品。
7. 简单筛选与总览统计。
8. 基础本地保存能力。
9. V0 手动测试清单、文档同步和收尾。

## 当前小阶段

### V0-5：编辑和删除食品

状态：准备开始。

目标：在当前页面内存数据基础上，增加最小可用的编辑和删除能力，让用户可以修改或移除已经新增或已有的食品。

本阶段只做：

- 在食品详情或卡片中提供明显的“编辑”入口。
- 用户可以编辑当前食品的基础字段。
- 编辑后继续复用日期计算、排序和状态工具。
- 编辑后详情和列表展示同步更新。
- 在食品详情或卡片中提供明显的“删除”入口。
- 删除前必须有确认提示，避免误删。
- 删除后从当前页面内存列表移除该食品。
- 保持老年友好：按钮明显、提示清楚、不要依赖隐藏手势。

本阶段暂不做：

- 本地保存。
- 云数据库。
- 识别能力。
- 提醒调度。
- 新依赖引入。
- 批量编辑或批量删除。
- 复杂页面跳转架构。
