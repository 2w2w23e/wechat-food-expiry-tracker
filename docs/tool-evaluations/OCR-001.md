# OCR-001 视频生产日期离线评测

状态：工具骨架已建立，真实样本评测待用户把视频放入本地 `video/` 目录后执行。

## 目标

先在开发机对本地视频样本做抽帧和 OCR 评测，找出生产日期、保质期、到期日等候选字段。该节点不进入 APK，不写食品数据，不做自动保存。

## 工具位置

```text
tools/ocr_eval/
```

本地样本和输出均被忽略：

```text
video/
.local/ocr-eval/OCR-001/
tools/ocr_eval/out/
tools/ocr_eval/models/
```

## 推荐执行

```powershell
python tools/ocr_eval/run_ocr_eval.py --input video --out .local/ocr-eval/OCR-001 --engine tesseract
```

无外部依赖自检：

```powershell
python tools/ocr_eval/run_ocr_eval.py --self-test
```

## 输出

- `manifest.json`
- `frames/`
- `ocr_raw.jsonl`
- `ocr_raw.txt`
- `candidates.jsonl`
- `metrics.csv`
- `report.html`

## 后续评测矩阵

| 引擎 | 用途 | 备注 |
| --- | --- | --- |
| FFmpeg | 抽帧 | 必需外部工具 |
| Tesseract `chi_sim+eng` | baseline OCR | 本工具先支持 |
| PaddleOCR | 中文包装强候选 | 后续作为精度对照 |
| EasyOCR | 备用对照 | 依赖较重 |

## 验收标准

- 不提交真实视频、抽帧、OCR 输出或模型缓存。
- `report.html` 能看到关键帧、raw OCR 和 candidateOnly 字段。
- `candidates.jsonl` 中所有结构化字段都标记为候选，不写入 APK 数据。
- 后续进入 APK 实时识别前，必须设计用户确认页。
